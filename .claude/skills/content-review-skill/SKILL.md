# 内容审核技能

## Kevin 的写作风格

### 语言特点
- 直接简洁，快速切入主题
- 第一人称，用"我"
- 适度口语化
- 避免过度修饰
- 技术术语保留英文

### 典型句式
✓ "我花了三年才想明白..."
✓ "这个方法帮我节省了50%的时间"
✓ "说实话，一开始我也..."
✓ "如果你也遇到XX问题..."

❌ "笔者认为..."
❌ "众所周知..."

## 结构审核

### 开头（10-15%）
- 快速说明主题
- 给出阅读理由
- 可选：分享个人背景

### 正文（70-80%）
- 逻辑清晰，有章节
- 每个观点有支撑
- 段落3-5行
- 使用小标题

### 结尾（10-15%）
- 总结核心观点
- 可选：行动建议或思考问题

## 语言质量检查

**清晰度**
- ❌ "这个工具的使用体验还是比较不错的"
- ✓ "这个工具帮我节省了30%的编码时间"

**具体性**
- ❌ "经过一段时间的努力"
- ✓ "经过3个月的练习"

**真实性**
- ✓ 承认困难和失败
- ✓ 分享真实数据
- ❌ 过度美化结果

## 可读性优化
- 段落不超过5行
- 避免连续3个以上长句
- 使用列表呈现要点
- 适当使用emoji（每篇2-3个）

- [ ] 段落长度适中
- [ ] 语言简洁直接

- [ ] 结尾有总结
- [ ] 符合 Kevin 的风格
