# 咖啡角板块重新设计 - 任务规划

**创建时间**: 2026-01-09
**项目**: Personal Blog - Coffee Corner Redesign
**目标**: 重新设计咖啡角板块，实现与 Notion 数据库集成，增强用户体验

---

## 项目目标

重新设计咖啡角板块前端，使其：
1. **简化导航层级** - 从 3 次点击减少到 1-2 次点击
2. **扁平化信息架构** - 单页面展示所有核心内容
3. **保持静态 HTML** - 使用现有的 Notion 同步脚本
4. **优化视觉设计** - 改进布局和交互体验
5. **保持 Neo-Brutalism 风格** - 与项目整体设计一致
6. **本地开发测试** - 在开发服务器中迭代优化

### Phase 2: 前端架构重新设计 🔄 进行中
**目标**: 设计扁平化、单页面咖啡角体验

**设计方案**:

**选项 A: 标签页单页设计** ⭐ 推荐
```
coffee.html (单页应用)
├── 固定顶部: 标题 + 统计
├── Tab 导航: [器具] [豆子] [探店] [日记]
└── 内容区: 所有内容垂直展示
    ├── 器具区 (展开卡片，点击展开详情)
    ├── 豆子区 (网格卡片)
    ├── 探店区 (网格卡片)
    └── 日记区 (时间线)
```

优点:
- ✅ 1次点击到达任何内容
- ✅ 所有内容在主页，清晰直观
- ✅ 保留原 HTML，无需重构
- ✅ 搜索方便（Ctrl+F）

**选项 B: 折叠式手风琴设计**
```
coffee.html (长滚动页面)
├── Hero Section
├── 折叠区块 1: 器具收藏 (可展开/收起)
│   └── 内嵌详情展开
├── 折叠区块 2: 豆子档案
├── 折叠区块 3: 探店笔记
└── 折叠区块 4: 冲煮日记
```

优点:
- ✅ 渐进式信息展示
- ✅ 移动端友好
- ⚠️ 需要大量 JavaScript
